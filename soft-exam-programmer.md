## 一、计算机组成与结构

### 1.1 CPU结构与指令系统

**考点**:
- **CPU组成**: 运算器(ALU、AC、MQ、DR) + 控制器(PC、IR、AR、ID)
- **指令格式**: 操作码 + 地址码
  - 操作码：定长/扩展编码
  - 地址码：零地址、一地址、二地址、三地址
- **寻址方式**: 立即寻址、直接寻址、间接寻址、相对寻址、基址寻址、变址寻址

**常考计算题**:
- 指令流水线计算：
  - 吞吐率：TP = n / (k + n - 1)Δt （n条指令，k级流水线）
  - 加速比：S = (n * k * Δt) / ((k + n - 1)Δt) = (nk)/(k+n-1)
  - 效率：E = (n * k * Δt) / (m * (k + n - 1)Δt) （m是流水线段数）

**例**: 4级流水线，完成10条指令：
- 周期：(4 + 10 - 1) = 13Δt
- 吞吐率：10 / 13Δt ≈ 0.77/Δt
- 加速比：(10 × 4) / 13 ≈ 3.08

### 1.2 存储器层次结构

**考点**:
- **Cache命中率**: 平均存取时间 = 命中率 × Cache存取时间 + (1-命中率) × 主存存取时间
- **主存编址**: 存储容量计算，按字节编址/按字编址
  - 例：地址线A15~A0（16根），数据线D7~D7（8根）→ 容量 = 2¹⁶ × 8位 = 64KB
- **Cache映射**: 直接映射、全相联、组相联
  - 直接映射：主存块号 → Cache块号 = 主存块号 mod Cache行数
- **虚拟内存**: 页式、段式、段页式，页表、快表(TLB)

### 1.3 中断与I/O

**考点**:
- **I/O控制方式**:
  - 程序查询方式：CPU全程等待，效率低
  - 中断方式：CPU与设备并行，每个字/字节中断一次
  - DMA方式：整块数据传送，DMA控制器接管总线，中断只在完成时
  - 通道方式：独立硬件，多个设备共享
- **中断优先级**: 硬件排队、软件判优，中断嵌套

### 1.4 流水线性能计算（必考题）

| 指标 | 公式 |
|------|------|
| 完成时间 | T = (k + n - 1)Δt |
| 吞吐率 | TP = n / T |
| 加速比 | S = (n × k × Δt) / T |
| 效率 | E = S / k |

## 二、操作系统

### 2.1 进程管理

**考点**:
- **三态模型**: 运行 → 就绪（时间片到）、运行 → 等待（I/O）、等待 → 就绪（I/O完成）
- **进程同步**: 临界区、信号量P/V操作
  - P操作：S减1，若S<0则阻塞等待
  - V操作：S加1，若S≤0则唤醒一个阻塞进程
  - 互斥：初值=1；同步：初值=0
- **死锁**:
  - 四个必要条件：互斥、占有并等待、非剥夺、循环等待
  - 银行家算法：安全状态判断，避免死锁
  - 死锁检测与解除：资源剥夺、进程回退、进程终止

**银行家算法思路**:
1. 计算剩余可用资源
2. 找一个进程，它的请求 ≤ 剩余可用
3. 假设分配，回收其全部资源
4. 重复直到所有进程都能完成 → 安全，否则不安全

### 2.2 内存管理

**考点**:
- **页式存储**: 逻辑地址 = 页号 + 页内地址，物理地址 = 物理块号 × 页大小 + 页内地址
- **页面置换算法**:
  - FIFO：先进先出，会出现Belady异常
  - LRU：最近最少使用，性能最好
  - OPT：最佳置换，理论最优，无法实现
- **分段vs分页**:
  - 分页：页是物理单位，大小固定，方便存储管理
  - 分段：段是逻辑单位，大小不固定，方便用户编程

### 2.3 文件系统与设备管理

**考点**:
- **文件目录**: 一级目录、二级目录、树形目录、无环图目录
- **磁盘调度**:
  - FCFS：先来先服务
  - SSTF：最短寻道时间优先
  - SCAN（电梯算法）：沿一个方向走到头再折返
  - C-SCAN：循环扫描，到头从另一端开始
- **Spooling技术**: 假脱机，将低速I/O设备改造成共享设备，用磁盘模拟脱机输入输出

## 三、程序设计语言和编译原理

### 3.1 文法与有限自动机

**考点**:
- **文法分类**:
  - 0型（短语文法）：能力最强，图灵机
  - 1型（上下文有关）：每产生式α→β满足|α| ≤ |β|
  - 2型（上下文无关）：左部单个非终结符 → 语法分析
  - 3型（正规文法）：右线性/左线性 → 词法分析
- **推导**: 最左推导（每次替换最左非终结符）、最右推导
- **有限自动机**:
  - NFA：非确定，一个状态对同一输入可以有多条转移
  - DFA：确定，每个状态每个输入至多一个转移
  - 可以转化：NFA → 子集法 → DFA
  - 考查：判断某个字符串是否能被接受

**识别字符串步骤**:
1. 从初态出发
2. 按字符串顺序依次读字符，跟随状态转移
3. 读完后停在终态 → 接受，否则不接受

### 3.2 编译过程

| 阶段 | 功能 | 产出 |
|------|------|------|
| 词法分析 | 分割单词，识别记号 | 单词符号流 |
| 语法分析 | 根据语法分析句子结构，生成语法树 | 语法树/抽象语法树 |
| 语义分析 | 分析语义，收集类型信息，类型检查 | 中间代码 |
| 中间代码生成 | 生成与机器无关的中间代码 | 四元式/三元式/树 |
| 代码优化 | 优化中间代码 | 优化后的中间代码 |
| 目标代码生成 | 生成机器语言/汇编 | 目标代码 |

### 3.3 表达式求值

- **前缀**: 运算符在前面，栈：从右往左扫描，遇操作符弹出两个数运算
- **中缀**: 运算符在中间，需要括号
- **后缀（逆波兰）**: 运算符在后面，栈：从左往右扫描，遇操作符弹出两个数运算

**例**: (a+b)*(c-d) → 后缀：ab+cd-*

## 四、数据结构

### 4.1 线性表

**考点**:
- **数组**: 存储地址计算，二维数组按行/按列存储
  - 行优先：Loc = base + (i×n + j) × size
  - 列优先：Loc = base + (j×m + i) × size
- **链表**: 单链表、双向链表、循环链表，插入删除操作指针修改
- **栈**: 后进先出，出栈序列合法性判断
- **队列**: 先进先出，循环队列判断空/满：牺牲一个单元，(rear+1)%n=front表示满

### 4.2 树和二叉树

**重要公式**:
- **二叉树第i层**: 最多 2^(i-1) 个结点
- **深度为k的二叉树**: 最多 2^k - 1 个结点
- **n个结点的完全二叉树**: 深度 = ⌊log₂n⌋ + 1
- **叶子结点数n₀ = 度为2结点数n₂ + 1** → n₀ = n₂ + 1（常考！）
- **完全二叉树**: 编号i，父结点⌊i/2⌋，左孩子2i，右孩子2i+1

**遍历**:
- 先序：根 → 左 → 右
- 中序：左 → 根 → 右
- 后序：左 → 右 → 根
- 层序：从上到下，从左到右
- **已知两种遍历求第三种**:
  - 先序+中序 或 后序+中序 → 唯一确定二叉树
  - 先序找根，中序分左右，递归构建

**二叉查找树（BST）**:
- 左子树所有结点 < 根，右子树所有结点 > 根
- 中序遍历得到有序序列
- 插入：找叶子位置插入；删除：三种情况，找后继替换

**平衡二叉树（AVL）**:
- 左右子树高度差绝对值 ≤ 1
- 旋转：LL、RR、LR、RL四种调整

**哈夫曼树**:
- 带权路径长度WPL最小的二叉树
- n个叶子 → 总结点数 = 2n-1，没有度为1的结点
- WPL = Σ(权值 × 路径长度)

### 4.3 图

**考点**:
- **存储**: 邻接矩阵（稠密图）、邻接表（稀疏图）
- **遍历**:
  - DFS深度优先：类似先序遍历，栈/递归
  - BFS广度优先：层序，队列
- **拓扑排序**: 有向无环图，每次删入度为0结点，输出，不唯一
- **最短路径**:
  - Dijkstra：单源，所有边权非负，每次选最近点，松弛，O(n²)
  - Floyd：所有点对，动态规划，O(n³)
- **最小生成树**:
  - Prim：从点开始，每次选最近点加入，O(n²)，适合稠密图
  - Kruskal：按权从小到大选边，不构成环就加，适合稀疏图

### 4.4 排序

**时间空间复杂度总结（必背）**:

| 排序算法 | 最好 | 最坏 | 平均 | 空间 | 稳定性 |
|---------|------|------|------|------|--------|
| 直接插入 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 冒泡 | O(n) | O(n²) | O(n²) | O(1) | 稳定 |
| 简单选择 | O(n²) | O(n²) | O(n²) | O(1) | 不稳定 |
| 希尔 | - | - | O(n^1.3) | O(1) | 不稳定 |
| 快速 | O(nlogn) | O(n²) | O(nlogn) | O(logn)~O(n) | 不稳定 |
| 堆排序 | O(nlogn) | O(nlogn) | O(nlogn) | O(1) | 不稳定 |
| 归并 | O(nlogn) | O(nlogn) | O(nlogn) | O(n) | 稳定 |

**口诀**: 选快希堆都不稳，插冒归并是稳定

**考点**:
- 快排：基准选择，分治，递归
- 堆排：大顶堆/小顶堆，建堆，调整，取出堆顶
- 稳定排序：相等元素原顺序保持不变

### 4.5 查找

- **二分查找**: 有序顺序表，O(logn)，判定树，平均查找长度
- **散列表**: 哈希函数，冲突处理（开放地址法、链地址法），装填因子 = 元素数 / 表长

## 五、软件工程

### 5.1 软件开发模型

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| 瀑布模型 | 线性顺序，文档驱动，阶段划分清晰 | 需求明确的项目 |
| 原型模型 | 快速构建原型，用户反馈修改 | 需求不明确、探索型 |
| 螺旋模型 | 瀑布+原型+风险分析，四个阶段 | 大型复杂项目，风险高 |
| 增量模型 | 增量交付，每个增量可运行 | 需求逐步明确，快速交付 |
| 喷泉模型 | 迭代，无间隙，面向对象开发 | 面向对象项目 |
| V模型 | 测试和开发阶段对应 | 强调测试提前 |
| 敏捷开发 | 迭代，用户参与，响应变化，拥抱变化 | 需求变化快，小团队 |

### 5.2 需求分析

- **需求分类**: 功能需求、非功能需求（性能、可靠性、可用性...）、设计约束
- **需求分析产出**: 需求规格说明SRS
- **DFD数据流图**: 实体（矩形）、加工（圆/圆角矩形）、数据存储（矩形两边平行线）、数据流（箭头）

### 5.3 软件设计

- **模块化**: 高内聚低耦合
  - 内聚：巧合 → 逻辑 → 时间 → 过程 → 通信 → 顺序 → 功能（↑越来越好）
  - 耦合：非直接 → 数据 → 标记 → 控制 → 公共 → 内容（↓越来越好）
- **结构化设计**: 面向数据流，变换流/事务流，变换分析设计模块结构
- **面向对象设计**: 五大原则SOLID
  - 单一职责：一个类只做一件事
  - 开闭：对扩展开放，对修改关闭
  - 里氏替换：子类可以替换父类
  - 依赖倒置：依赖抽象不依赖具体
  - 接口隔离：多个小接口好过一个大接口

### 5.4 软件测试

- **测试阶段**: 单元测试 → 集成测试 → 确认测试 → 系统测试
  - 单元：模块内部，白盒
  - 集成：模块间接口，黑盒+白盒
  - 确认：验证需求，黑盒
  - 系统：整个系统，包括软硬件，黑盒
- **测试方法**:
  - 白盒：覆盖路径，语句覆盖、判定覆盖、条件覆盖、判定条件覆盖、条件组合覆盖、路径覆盖（覆盖强度↑）
  - 黑盒：等价类划分、边界值分析、错误推测、因果图
- **McCabe环路复杂度**:
  - V(G) = E - N + 2 （E边数，N结点数）
  - V(G) = P + 1 （P判定结点数）
  - V(G) = 区域数（包括外部区域）

### 5.5 软件维护

- **维护分类**: 改正性、适应性、完善性、预防性
- **可维护性**: 可理解性、可测试性、可修改性
- **文档**: 用户文档、系统文档

## 六、面向对象技术

### 6.1 UML图

**常考**:
- **用例图**: 参与者、用例、关系（包含include、扩展extend、泛化）
  - include：基用例必须执行被包含用例
  - extend：扩展用例可选扩展基用例
- **类图**: 类之间关系
  - 依赖（虚线箭头）：一个类用到另一个，"uses a"
  - 关联（实线）：结构关系，"has a"，多重度：1、*、0..1、1..*
  - 聚合（空心菱形实线）：整体和部分，可分离，"has a"
  - 组合（实心菱形实线）：整体和部分，不可分离，同生命周期，"contains a"
  - 泛化（空心箭头实线）：继承，"is a"
  - 实现（空心箭头虚线）：接口实现
- **顺序图**: 对象、生命线、消息，交互顺序
- **状态图**: 状态、转移、事件，对象生命周期

### 6.2 设计模式

**分类**:
- 创建型：工厂方法、抽象工厂、单例、建造者、原型
- 结构型：适配器、桥接、组合、装饰、外观、享元、代理
- 行为型：模板方法、策略、命令、观察者、迭代器、中介者、备忘录、解释器、状态、访问者、责任链、策略

**常考设计模式**:

| 模式 | 意图 |
|------|------|
| **单例** | 保证一个类只有一个实例，提供全局访问点 |
| **工厂方法** | 定义接口创建对象，让子类决定实例化哪个类 |
| **抽象工厂** | 创建一系列相关对象，不指定具体类 |
| **适配器** | 将一个类接口转换成客户期望的另一个接口，解决接口不兼容 |
| **装饰器** | 动态给对象添加额外职责，比继承更灵活 |
| **观察者** | 定义对象间一对多依赖，一个对象改变通知所有依赖者自动更新 |
| **策略** | 定义一系列算法，封装每个，可互相替换，算法独立于客户变化 |
| **模板方法** | 定义算法骨架，子类重定义算法某些步骤，不改变结构 |

## 七、数据库

### 7.1 ER模型

- **实体**: 矩形，**联系**: 菱形，**属性**: 椭圆
- **联系类型**: 1:1、1:n、m:n
- **转换规则**:
  - 实体转关系：属性就是关系的属性，主键就是实体主键
  - 1:1联系：可以转独立关系，或合并到任意一端
  - 1:n联系：可以转独立关系，或合并到n端
  - m:n联系：必须转独立关系，属性是双方主键+联系自身属性

### 7.2 关系代数

基本运算：
- 选择σ：选满足条件的行
- 投影π：选需要的列
- 笛卡尔积×：两个关系拼接，所有组合
- 自然连接⋈：自动比较相同属性，相等就连接，去掉重复列

### 7.3 规范化理论

**函数依赖**:
- 完全函数依赖：X→Y，X的任何真子集都不能决定Y
- 部分函数依赖：X→Y，存在X的真子集能决定Y
- 传递依赖：X→Y，Y→Z，Y不→X，则X→Z传递

**范式**:
- **1NF**: 属性不可再分
- **2NF**: 消除非主属性对码的部分依赖
- **3NF**: 消除非主属性对码的传递依赖
  - 结论：每一个非主属性既不部分依赖于码，也不传递依赖于码
  - 3NF分解：保持函数依赖，无损连接
- **BCNF**: 消除主属性对码的部分和传递依赖，所有决定因素都包含码

**候选码求法**:
1. 找出只在左边出现的属性 → 一定在候选码中
2. 找出只在右边出现的属性 → 一定不在候选码中
3. 两边都没出现的 → 一定在候选码中
4. 闭包运算看能否覆盖所有属性

### 7.4 SQL语法

**常考结构**:
```sql
-- 基本查询
SELECT 列 FROM 表 WHERE 条件 GROUP BY 列 HAVING 条件 ORDER BY 列;

-- 连接查询
SELECT * FROM A JOIN B ON A.id = B.a_id;

-- 插入更新删除
INSERT INTO 表(列...) VALUES(值...);
UPDATE 表 SET 列=值 WHERE 条件;
DELETE FROM 表 WHERE 条件;

-- 创建表
CREATE TABLE 表 (列 类型 PRIMARY KEY, ...);

-- 索引
CREATE INDEX idx_name ON 表(列);
```

### 7.5 事务与并发控制

- **事务ACID**: 原子性、一致性、隔离性、持久性
- **并发问题**:
  - 丢失更新：两个事务同时改，一个覆盖另一个
  - 不可重复读：同一事务两次读同一数据，中间被另一个改了，结果不同
  - 读脏数据：事务修改了数据，回滚了，另一个事务读到了这个修改
- **隔离级别**:
  - 读未提交：能读到未提交数据 → 都不能解决
  - 读已提交：只能读到已提交 → 解决脏读
  - 可重复读 → 解决脏读、不可重复读
  - 串行化 → 都解决

## 八、计算机网络

### 8.1 OSI七层模型

| 层 | 主要设备/协议 | 功能 |
|---|-------------|------|
| 物理层 | 集线器、中继器 | 传输二进制比特 |
| 数据链路层 | 交换机、网桥 | 帧封装、MAC地址、差错控制 |
| 网络层 | 路由器 | 路由选择、IP地址、拥塞控制 |
| 传输层 | 防火墙 | 端到端连接，TCP/UDP |
| 会话层 | - | 建立管理会话 |
| 表示层 | - | 数据格式转换、加密压缩 |
| 应用层 | 网关 | 用户接口，应用服务 |

### 8.2 TCP/IP协议簇

- **TCP**: 面向连接，可靠，字节流，三次握手建立，四次挥手释放 → 适合对可靠性要求高
- **UDP**: 无连接，不可靠，开销小 → 适合实时传输（视频、语音）
- **常用端口**:
  - 21：FTP
  - 23：Telnet
  - 25：SMTP
  - 53：DNS
  - 80：HTTP
  - 110：POP3
  - 443：HTTPS

### 8.3 IP地址

- **分类**:
  - A：1-126，默认掩码8位 → 网络位第一个字节，0开头（0xxxxxxx）
  - B：128-191，默认掩码16位 → 10开头（10xxxxxx）
  - C：192-223，默认掩码24位 → 110开头（110xxxxx）
  - D：224-239，组播
  - E：240-255，保留
- **CIDR**: 网络前缀，例如/24表示前24位网络位 → 主机位32-24=8位 → 地址数2^8=256，可用254（网络地址和广播地址）
- **计算网络地址**: IP 和 掩码 按位与

例：192.168.1.100/26 → 掩码255.255.255.192 → 192.168.1.64是网络地址

### 9.2 典型动态规划问题

**0-1背包**:
- n个物品，重量w[i]，价值v[i]，容量W，最大价值
- dp[i][j] = max(dp[i-1][j], dp[i-1][j-w[i]] + v[i])

**最长公共子序列**:
- 两个序列A、B，找最长公共子序列（不要求连续）
- dp[i][j] = if A[i] == B[j]: dp[i-1][j-1]+1 else: max(dp[i-1][j], dp[i][j-1])

**最长递增子序列**:
- 一个序列，找最长递增子序列长度
- O(n²)：dp[i] = max(dp[j]+1 for j<i if a[j]<a[i])
- O(nlogn)：二分维护 tails 数组
